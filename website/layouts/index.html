Why tenantplane

🔍

Explainable sync

Deterministic virtual-to-host mapping with dry-run planning. Ask why any host object exists and get an answer.

🛡️

Explicit isolation

Isolation profiles compose NetworkPolicy, ResourceQuota, LimitRange, and Pod Security into one auditable boundary.

🪶

Lightweight control planes

Each tenant runs a small k3s control plane in a host namespace — easy to understand, audit, and extend.

🔁

Day-2 by design

Reconciliation, drift-aware sync, and decision records built for the operational realities of multi-tenancy.

🧭

Migration paths

Evolve tenants across shared, dedicated, and private isolation models without recreating tenant API state.

📈

Observable

Decisions as Kubernetes Events today; OpenTelemetry and Prometheus on the roadmap.

Three resources, one mental model

TenantCluster

The lifecycle of one virtual Kubernetes tenant — shared, dedicated, or private.

IsolationProfile

The security and fairness boundary applied around a tenant's workloads.

SyncPolicy

Which resources cross the virtual-to-host boundary, and how conflicts resolve.
